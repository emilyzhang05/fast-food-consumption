# Fast Food Consumption Analysis

SQL, MongoDB, and Tableau analysis exploring how fast food consumption relates to nutrition, obesity risk, happiness, and physical activity trends across countries.

## Overview

This project analyzes fast food and nutrition datasets using both relational and NoSQL approaches. The analysis compares country-level fast food consumption rankings with dietary intake, overweight prevalence, happiness indicators, and physical activity profiles to identify evidence-based health risk patterns.

The project was completed as part of the BC2402 group project and includes SQL queries, MongoDB aggregation pipelines, raw datasets, and the final project report.

## Key Questions

- Which countries rank highest in fast food consumption and revenue?
- How do fast food consumption patterns compare with fat, carbohydrate, and protein intake?
- Is higher fast food consumption associated with overweight prevalence or other health risks?
- How do happiness and physical activity indicators relate to fast food consumption?
- Which fast food items or menu categories show higher nutritional risk?

## Tools Used

- SQL for relational querying, joins, grouping, filtering, and nutritional comparisons
- MongoDB for NoSQL aggregation pipelines and document-based analysis
- Tableau for dashboarding and stakeholder-facing visualizations
- CSV and JSON datasets for country-level and menu-level analysis


## Included Files

- `sql/Final_Group_Project_Team2.sql` contains the SQL analysis queries.
- `scripts/Final Group Project_Team 2_Sem9_2025.js` contains the MongoDB/NoSQL analysis.
- `data/raw/` contains the raw datasets used for the analysis.
- `docs/Bc2402 Sem 9 Group 2 Group Project Report.pdf` contains the final written report.

## Analysis Highlights

- Compared fast food consumption rankings with average macronutrient intake.
- Joined country-level fast food data with overweight prevalence indicators.
- Explored whether physical activity levels could help mitigate fast-food-related health risks.
- Ranked food items and categories using nutrition indicators such as calories, fat, sodium, cholesterol, protein, and health scores.
- Implemented equivalent analysis logic in both SQL and MongoDB to compare relational and NoSQL workflows.
